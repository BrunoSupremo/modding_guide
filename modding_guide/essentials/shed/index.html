How to save time with SHED

What is SHED?

SHED is short for Stonehearth Editor. It is a mod editor in progress. You can find the source code at the Stonehearth Github page, but you will need the Visual Studio IDE to compile it; we don't offer any executable yet since it has bugs and it isn't complete. It is written in C# using WinForms, although some of its editors are made with CEF.

Setup SHED

To get SHED up and running, follow these steps:

  1. Get a Visual Studio version (minimum VS 2014). If you have Unity, you might have installed already the Community version of Visual Studio, which is free. Otherwise you can find it here. This IDE takes very long to download and install, so be sure you want to use SHED despite its bugs (you can always make your mods with any text editor).
  2. Download the SHED code from here. Click in the "Clone or Download" button and save it in some folder in your computer.
  3. Open Visual Studio and go to File->Open->Project/Solution. Find the StonehearthEditor.sln file where you downloaded the code.
  4. Select either Debug or Release and x86 or 64 bit (depending on your computer's architecture). Click on the Start button or in Build->Build Solution.
  5. Once it has finished compiling, you can either run it from VS with the Start button, or find the exe in the respective folder.

Additional steps

If you want to use the auto-localization feature (which automatically prepares the translation file for your mod so that it can be translated to other languages), you must install Python 2.7 in your computer. Make sure it is that version and not Python 3. Then make sure you have the Python path in your environment variables so that the computer knows where to find it.

First time run

The first time you run SHED it will ask you to find your mods folder. Select the folder itself (or whatever folder you have your mods in).
Now, to be able to see and create any content in SHED, the mods must be in plain folders, not in .smod or .zip, so at this point you should manually rename your smods into zips and uncompress them in your mods folder. Make sure the structure is the same than when we run uncompressed mods in the game, otherwise SHED won't know how to load them.

Overview

Let's take a look at the interface of SHED. The options in the main menu are:

File:

  • New Mod
  • Save All
  • Reload
  • Change Mod Directory

View:

  • Net Worth Visualizer

Then we have the different tabs:

  • Manifest
  • Encounter Designer
  • Entity Stats Browser
  • Effects Editor
  • Recipe Editor
